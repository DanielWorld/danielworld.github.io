Java GC Generation

Header description

  • Young Generation
  • Old generation
  • Permanent generation

Usually, (young + old) generation is heap memory area.

1. Young generation (Eden space) 

: All new objects are first created in Eden space.

: Once space reaches then, Minor GC kicks in.

: First it removes all the non-referenced objects and move referenced objects from the 'eden' and 'from' into the 'to' survivor space.  

: Once the GC is over, the 'from' and 'to' roles are swapped.

1-1. Survivor1 (From) : 'to' role during the previous GC. 

1-2. Survivor2 (To) : During GC, that all referenced objects are moved to, from 'from' and 'eden'.

2. Old generation (Tenured) 

: Objects have their own age. (Age is the number of times that it has moved within the survivor space.)  

: Depending on the threshold limit, objects moved from the 'to' survivor space to Old generation space.

: When the heap is full or the old generation is full,  'Major GC' kicks in. ('Stop-the-World' happens here)   (Remember this, 'Minor GC' kicks in Young generation 'Major GC' kicks in Old generation)    

: There is another type of GC named 'Full GC' which involves other memory areas such as the permgen space. 

3. Permanent generation (Permgen space) 

: Store the following information   

1. Constant Pool (Memory Pool)

2. Field & Method Data and Code
